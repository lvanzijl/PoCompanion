# NSwag API Client Generation

This document describes how to regenerate the C# API client from the OpenAPI specification.

## Overview

The PoTool.Client project uses NSwag to generate a strongly-typed C# client from the OpenAPI specification. The generated client is located at:
```
PoTool.Client/ApiClient/ApiClient.g.cs
```

## Prerequisites

- .NET 10 SDK installed
- NSwag CLI tool installed (see Installation below)
- OpenAPI specification generated (see [OPENAPI.md](./OPENAPI.md))

## Installation

### Install NSwag as a local tool

From the repository root:

```powershell
# Create local tool manifest (if not exists)
dotnet new tool-manifest

# Install NSwag
dotnet tool install NSwag.ConsoleCore

# Restore tools (for fresh clones)
dotnet tool restore
```

This ensures all developers use the same NSwag version.

## Client Generation

### Step 1: Generate OpenAPI specification

First, ensure you have a current OpenAPI spec:

```powershell
.\tools\generate-openapi.ps1
```

This creates `PoTool.Client/openapi.json`.

### Step 2: Run NSwag

From the `PoTool.Client` directory:

```powershell
cd PoTool.Client
dotnet nswag run nswag.json
```

Or use the shorthand:

```powershell
cd PoTool.Client
nswag run nswag.json
```

This will:
1. Read the OpenAPI spec from `PoTool.Client/openapi.json`
2. Generate the C# client code
3. Save it to `PoTool.Client/ApiClient/ApiClient.g.cs`

### Step 3: Verify the build

```powershell
dotnet build PoTool.Client/PoTool.Client.csproj
```

The client should compile without errors.

## Configuration

The NSwag configuration is in `PoTool.Client/nswag.json`. Key settings:

- **Input**: `PoTool.Client/openapi.json` (generated by the API)
- **Output**: `ApiClient/ApiClient.g.cs`
- **Namespace**: `PoTool.Client.ApiClient`
- **Client naming**: `{controller}Client` (e.g., `ProfilesClient`, `WorkItemsClient`)
- **JSON library**: System.Text.Json
- **Nullable reference types**: Enabled

## Workflow

The complete workflow for updating the API client after API changes:

```powershell
# 1. Make changes to API controllers/DTOs
# 2. Build the API
dotnet build PoTool.Api

# 3. Generate fresh OpenAPI spec (starts API, downloads spec, stops API)
.\tools\generate-openapi.ps1

# 4. Add operation IDs to the OpenAPI spec
.\tools\add-operation-ids.ps1

# 5. Fix type and nullability issues in the OpenAPI spec
.\tools\fix-openapi-types.ps1

# 6. Regenerate the client
cd PoTool.Client
dotnet nswag run nswag.json
cd ..

# 7. Build and test the client
dotnet build PoTool.Client
dotnet test
```

**Note**: API client generation is **manual only**. The client is NOT automatically regenerated during build.
This ensures predictable builds and avoids issues with automatic generation during compilation.

## Troubleshooting

### "nswag: command not found"

Install NSwag as a local tool:

```powershell
dotnet tool install NSwag.ConsoleCore
dotnet tool restore
```

### "Input file not found"

Ensure you've generated the OpenAPI spec first:

```powershell
.\tools\generate-openapi.ps1
```

The file should exist at `PoTool.Client/openapi.json`.

### Build errors after regeneration

Common causes:
1. **Breaking API changes**: Review the API changes and update consuming code
2. **Namespace conflicts**: Check for duplicate type names in DTOs
3. **Incompatible NSwag version**: Ensure all developers use the same tool version (via local tool manifest)

### The generated client is empty or missing endpoints

Check that:
1. The OpenAPI spec is valid JSON
2. The API controllers are properly decorated with `[ApiController]` and route attributes
3. The API is running in Development mode when generating the spec

## NSwag Configuration Details

The `nswag.json` file contains:

```json
{
  "runtime": "Net100",
  "documentGenerator": {
    "fromDocument": {
      "json": "openapi.json"
    }
  },
  "codeGenerators": {
    "openApiToCSharpClient": {
      "namespace": "PoTool.Client.ApiClient",
      "output": "ApiClient/ApiClient.g.cs",
      ...
    }
  }
}
```

The `json` path is relative to the `PoTool.Client` directory where `nswag.json` is located.
