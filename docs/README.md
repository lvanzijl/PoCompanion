# Documentation Index

This repository uses the following canonical documentation structure:

- `docs/rules/` — mirrored rule documents
- `docs/architecture/` — stable system and product-structure reference
- `docs/implementation/` — active plans and forward-looking execution docs
- `docs/analysis/` — audits, diagnostics, investigations, and exploratory validation
- `docs/reports/` — dated task outputs and implementation results
- `docs/archive/` — deprecated and historical material

## Key entry points

- Architecture overview: `docs/architecture/`
- Active plans: `docs/implementation/`
- Analysis and audits: `docs/analysis/`
- Task reports: `docs/reports/`
- Legacy ingestion history: `docs/archive/legacy-revision-ingestion/`

Historical notes remain in `docs/archive/` and selected `docs/analysis/` documents where a dated audit still has traceability value.
