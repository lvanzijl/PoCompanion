# UI Semantic Rules

## Core semantics

- Story points are the planning and delivery metric.
- Effort hours are the engineering workload metric.

## UI labeling rules

- Never display effort hours with a `pts` suffix.
- Story-point surfaces should use explicit story-point labels when the underlying metric is story-point based.
- Effort-hour surfaces should use explicit effort-hour labels when the underlying metric is effort based.

## Portfolio flow exception

- Portfolio flow remains effort-based until its model changes.
- Portfolio flow UI must keep effort-hour semantics explicit until a story-point model is implemented.
